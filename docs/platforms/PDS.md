# PDS.md — Спецификация протокола

## KBD System UART Protocol v1.0

---

# 1. Общая информация

## 1.1 Назначение

Данный документ определяет **битовую и байтовую спецификацию всех протоколов системы**, включая:

- UART Host протокол
- UART Panel протоколы
- маршрутизацию событий клавиш
- управление подсветкой
- управление USB состоянием

---

## 1.2 Область применения

Протокол используется для:

- взаимодействия Host ↔ MCU клавиатуры
- взаимодействия MCU ↔ дочерние панели
- передачи событий клавиш
- управления подсветкой
- управления состоянием USB HID

---

## 1.3 Базовые свойства

- Транспорт: UART
- Размер кадра: 1 байт
- Фрейминг: отсутствует (byte-stream)
- CRC: отсутствует
- Подтверждения: отсутствуют
- Порядок: FIFO на уровне UART гарантируется аппаратно

---

# 2. Общая модель кодирования

## 2.1 Унифицированная структура байта

Все кадры протокола используют единый формат:

```
Бит:   7    6    5    4    3    2    1    0
      +----+----+----+----+----+----+----+----+
      | F  | F  |           PAYLOAD            |
      +----+----+----+----+----+----+----+----+
```

---

## 2.2 Поля

|Поле|Биты|Описание|
|---|---|---|
|F[7:6]|2|Тип кадра / контекст|
|DATA[5:0]|6|Полезная нагрузка (код клавиши / яркость / индекс)|

---

## 2.3 Декодирование типа кадра

|F[7:6]|Тип|
|---|---|
|00|События клавиш (HOST формат)|
|01|События PANEL 1|
|10|События PANEL 2|
|11|Управляющие команды|

---

# 3. ПРОТОКОЛ КЛАВИШ (HOST DOMAIN)

---

## 3.1 Формат кадра клавиши

```
Бит:   7    6    5    4    3    2    1    0
      +----+----+----+----+----+----+----+----+
      | 0  | S  |        KEYCODE (6 бит)      |
      +----+----+----+----+----+----+----+----+
```

---

## 3.2 Поля

|Поле|Биты|Описание|
|---|---|---|
|F[7]|0|домен host-клавиш|
|S|6|состояние клавиши (1=нажата, 0=отпущена)|
|KEYCODE|0–5|логический код клавиши|

---

## 3.3 Семантика

|S|Значение|
|---|---|
|1|клавиша нажата|
|0|клавиша отпущена|

---

## 3.4 Повтор удержания

Если клавиша удерживается:

- кадр должен повторяться каждые **200 мс**
- KEYCODE не изменяется
- S = 1

---

# 4. ПРОТОКОЛ ЯРКОСТИ

---

## 4.1 Формат кадра

```
Бит:   7    6    5    4    3    2    1    0
      +----+----+----+----+----+----+----+----+
      | 1  | 1  |     ЯРКОСТЬ (6 бит)         |
      +----+----+----+----+----+----+----+----+
```

---

## 4.2 Диапазон

|Поле|Значение|
|---|---|
|яркость|0–15 (эффективный диапазон)|

---

## 4.3 Отображение

```
PWM = яркость / 15
```

---

# 5. УПРАВЛЯЮЩИЕ КОМАНДЫ

---

## 5.1 Отключение USB

```
0xEC = 1110 1100
```

---

## 5.2 Включение USB

```
0xED = 1110 1101
```

---

## 5.3 Семантика

|Команда|Действие|
|---|---|
|0xEC|отключить USB HID|
|0xED|включить USB HID|

---

## 5.4 Heartbeat

```
0x1F = 0001 1111
```

- отправляется раз в 1000 мс
- полезной нагрузки нет

---

# 6. ПРОТОКОЛ ПАНЕЛЕЙ

---

## 6.1 Формат кадра панели

```
Бит:   7    6    5    4    3    2    1    0
      +----+----+----+----+----+----+----+----+
      | P  | P  |     ЛОКАЛЬНЫЙ КОД (6 бит)   |
      +----+----+----+----+----+----+----+----+
```

---

## 6.2 Код панели

|F[7:6]|Панель|
|---|---|
|01|Панель 1|
|10|Панель 2|

---

## 6.3 Локальные клавиши

|Панель|Диапазон|
|---|---|
|Panel 1|1–7|
|Panel 2|1–7|

---

## 6.4 Преобразование в глобальные коды

### Панель 1

```
глобальный_код = локальный + 18
```

### Панель 2

```
глобальный_код = локальный + 25
```

---

# 7. ПРАВИЛА МАРШРУТИЗАЦИИ

---

## 7.1 Host → Panel

```
если key ∈ [19..25] → Panel 1
если key ∈ [26..32] → Panel 2
```

---

## 7.2 Panel → Host

```
если key > 18 → передать в Host
иначе → обновить матрицу клавиш
```

---

## 7.3 USB фильтрация

USB команды:

- НЕ передаются на панели
- обрабатываются только локально

---

## 7.4 Яркость

Команды яркости:

- всегда передаются без изменений
- не фильтруются

---

# 8. МОДЕЛЬ СОСТОЯНИЙ

---

## 8.1 Состояния клавиши

```
IDLE → PRESSED → RELEASED → IDLE
```

---

## 8.2 Антидребезг

Состояние клавиши считается подтверждённым только при стабильности сигнала **20 мс**.

---

# 9. МОДЕЛЬ ОШИБОК

---

## 9.1 Некорректные данные

- CRC отсутствует
- любые некорректные байты игнорируются или интерпретируются по контексту

---

## 9.2 Требование устойчивости

Система обязана:

- не падать от некорректных данных
- не терять синхронизацию при повреждении байта
- восстанавливаться на следующем валидном событии

---

# 10. ВРЕМЕННЫЕ ОГРАНИЧЕНИЯ

|Событие|Ограничение|
|---|---|
|антидребезг|20 мс|
|удержание клавиши|200 мс|
|heartbeat|1000 мс|
|обработка UART|< 2 мс|
|задержка USB|< 10 мс|

---

# 11. СОВМЕСТИМОСТЬ

---

## 11.1 Обратная совместимость

- сохранены legacy команды 0xEC / 0xED
- формат панелей фиксирован

---

## 11.2 Правила версии

При изменении протокола:

- нельзя менять KEYCODE семантику
- нельзя менять debounce без увеличения версии
- расширение только через F[7:6]
